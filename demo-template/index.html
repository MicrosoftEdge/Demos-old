    Required Components

    Nav bar

    This component appears at the very top of the page. It sets the context for the demo, and provides navigation back to other demos.

    Overrideable styles can be found in the COMPONENT: NAV BAR section of demo.css. Do not override anything other than these styles, as it is very important this bar feels consistent from demo to demo. If you think you have a good idea for a universal change to this bar, please file an issue. The mechanism to explore more demos is likely to change.

    demo Navigation

    This is the dropdown that lives in the nav bar. It is automagically populated with JavaScript based on the sections in your demo. For this to work, your section tags should be formed like this:

    
<section id="your-hash" data-nav-label="Title for Nav" class="l-section"></section>

    The ID is used for the window hash location (jump link). The data-nav-label is used for the title of this section in the demo navigation. This doesn't have to perfectly match the title of the section, it can be a shorter version if that makes sense for the nav.

    Intro

    The large area at the top of the demo. You might consider making this expand to fill the viewport height, but we have left that option open for you.

    Overrideable styles can be found in the COMPONENT: INTRO section of demo.css. It's pretty lightweight, so lots of flexibility here!

    Outro

    This is the bar at the bottom of the demo which provides credits for the demo, a link to Github, and will eventually be expanded to include related demos. Fill out the names and URLs (Github preferred) for the contributors, as well as the Github URL.

    Included Styles

    The demo template comes with very minimal styles, as we would like you to feel free to be creative. Here's what's included.

    Layout

    There are a couple layout classes, denoted by the namespace prefix l-:

    • .l-contain creates a container with a max width and horizontal paddings; it can be used inside or outside of sections (see below). Please do not override these paddings.
    • .l-section and .l-subsection add top margins, thereby creating page sections and subsections. If you would like to override the margins on these, you need to have a good reason for doing so.
    • .l-section--banner and .l-subsection--banner can be added onto those sectioning classes respectively. These add top and bottom paddings to match their top margins, as well as a light grey background for clarity (it is expected you'll probably override that background color).

    Body Copy

    Body copy has default styles, top margins, and maximum widths. If a body copy element matches the :first-child selector, the top margin is 0. If you change the body copy styles defined here, there should be a good reason to do so.

    Utilities

    Utilities are denoted by the u- namespace prefix.

    • .u-clear: place this class on a parent and it will clear its children's floats.
    • .u-sr-only: visibly hides content, without hiding the content from screen readers.
